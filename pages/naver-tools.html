네이버 웹마스터도구(Search Advisor)는 사이트의 검색 현황을 모니터링하고 최적화하는 필수 도구입니다. 수집요청, RSS/사이트맵 제출, URL 검사, SEO 리포트, 사이트 진단, IndexNow 프로토콜을 통해 네이버 검색에서 사이트의 가시성을 관리할 수 있습니다.

인용 가능 정의: 네이버 웹마스터도구(Search Advisor)는 사이트의 검색 수집·색인·노출 현황을 모니터링하고, RSS/사이트맵 제출, URL 검사, SEO 리포트 등 최적화 기능을 제공하는 도구입니다. [출처: 네이버 서치어드바이저, "웹마스터 가이드"]

1. 수집요청 및 검색제외

수집요청

웹마스터도구의 수집요청 기능을 사용하면 네이버 검색로봇 Yeti에게 특정 URL의 수집을 직접 요청할 수 있습니다. 새로 콘텐츠를 발행했거나 기존 콘텐츠를 대대적으로 수정한 경우 이 기능을 활용하세요.

ℹ️
수집요청 절차
  1. 네이버 웹마스터도구 → 수집현황 → 수집요청 메뉴로 이동
  2. 수집을 요청할 URL을 입력 (정확한 전체 URL 입력)
  3. 요청 버튼 클릭
  4. 수집 요청 후 Yeti 로봇이 해당 URL을 방문하여 수집을 진행

⚠️ 주의사항

  • 수집요청은 일일 요청 횟수 제한이 있습니다. 불필요한 요청은 자제하세요.
  • 수집을 요청했다고 해서 즉시 색인(검색 노출)이 보장되는 것은 아닙니다.
  • robots.txt에서 차단된 URL은 수집요청을 해도 수집되지 않습니다.
  • 수집요청 전 해당 URL이 정상적으로 접속 가능한 상태인지 확인하세요.

검색 제외

검색 결과에서 특정 페이지를 제외하려면 <meta name="robots" content="noindex"> 태그를 해당 페이지에 추가하거나, robots.txt에서 Yeti 로봇의 접근을 차단하면 됩니다.

<!-- HTML noindex 메타태그 -->
<meta name="robots" content="noindex">

<!-- 특정 로봇만 제외 -->
<meta name="Yeti" content="noindex">

<!-- robots.txt에서 제외 -->
User-agent: Yeti
Disallow: /private/

2. RSS 및 사이트맵 제출

RSS와 사이트맵을 제출하면 네이버 검색로봇이 사이트의 콘텐츠 구조를 빠르게 파악하고 새로운 콘텐츠를 신속하게 수집할 수 있습니다.

RSS 문서 예제

RSS(Really Simple Syndication)는 사이트의 새 콘텐츠를 검색엔진에 알리는 간단한 XML 형식입니다.

<?xml version="1.0" encoding="UTF-8"?>
<rss version="2.0">
  <channel>
    <title>사이트 제목</title>
    <link>https://example.com</link>
    <description>사이트 설명</description>
    <language>ko</language>
    <lastBuildDate>Mon, 17 Jun 2026 09:00:00 +0900</lastBuildDate>
    <item>
      <title>새 글 제목</title>
      <link>https://example.com/new-post</link>
      <description>새 글 요약</description>
      <pubDate>Mon, 17 Jun 2026 09:00:00 +0900</pubDate>
    </item>
  </channel>
</rss>

사이트맵(Sitemap)

사이트맵은 사이트의 모든 페이지 목록을 검색엔진에 제공하는 XML 파일입니다. 사이트맵 URL 목록과 사이트맵 인덱스 두 가지 방식을 지원합니다.

사이트맵 URL 목록 예제

<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://example.com/</loc>
    <lastmod>2026-06-17</lastmod>
    <changefreq>daily</changefreq>
    <priority>1.0</priority>
  </url>
  <url>
    <loc>https://example.com/page1</loc>
    <lastmod>2026-06-16</lastmod>
    <changefreq>weekly</changefreq>
    <priority>0.8</priority>
  </url>
</urlset>

사이트맵 인덱스 예제

사이트맵이 여러 개인 경우 사이트맵 인덱스 파일을 사용하여 하나로 묶어 제출할 수 있습니다.

<?xml version="1.0" encoding="UTF-8"?>
<sitemapindex xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <sitemap>
    <loc>https://example.com/sitemap-posts.xml</loc>
    <lastmod>2026-06-17</lastmod>
  </sitemap>
  <sitemap>
    <loc>https://example.com/sitemap-pages.xml</loc>
    <lastmod>2026-06-10</lastmod>
  </sitemap>
</sitemapindex>

제출 주의사항

⚠️ 중요 제한사항

  • 용량 제한: RSS 및 사이트맵 파일은 10MB를 초과할 수 없습니다.
  • URL 도메인 일치: 사이트맵에 포함된 모든 URL은 웹마스터도구에 등록된 사이트의 도메인과 일치해야 합니다.
  • 인코딩: UTF-8 인코딩을 사용해야 합니다.
  • 최대 URL 개수: 사이트맵 하나당 최대 50,000개의 URL을 포함할 수 있습니다.
  • 사이트맵은 웹마스터도구의 사이트맵 제출 메뉴에서 제출하거나, robots.txt에 사이트맵 위치를 명시할 수 있습니다.

3. 사이트 간단 체크

웹마스터도구의 사이트 간단 체크 기능을 사용하면 사이트의 기본적인 SEO 상태를 빠르게 진단할 수 있습니다.

검사 항목

항목설명
검색로봇 접근 확인네이버 검색로봇(Yeti)이 사이트에 정상적으로 접근할 수 있는지 확인합니다. 방화벽, 서버 설정, robots.txt 차단 여부를 점검합니다.
robots.txt 확인robots.txt 파일이 정상적으로 설정되어 있고, Yeti 로봇이 필요한 콘텐츠에 접근할 수 있는지 확인합니다.
title/description 확인각 페이지의 <title>과 <meta name="description">가 적절히 설정되어 있는지 확인합니다. 중복되거나 누락된 경우 경고를 표시합니다.
noindex 확인해당 페이지에 noindex 메타태그가 설정되어 있는지 확인합니다. 의도치 않게 검색 결과에서 제외된 페이지가 없는지 점검할 수 있습니다.

💡 활용 팁

사이트 간단 체크는 정기적으로 실행하여 사이트의 기본 SEO 상태를 모니터링하는 것이 좋습니다. 특히 사이트 구조를 변경하거나 대규모 업데이트 후에는 반드시 실행하세요.

4. URL 검사

웹마스터도구의 URL 검사 기능은 특정 URL에 대한 네이버 검색엔진의 상태를 상세히 분석합니다. 색인, Meta, SEO 세 가지 항목으로 나누어 검사 결과를 제공합니다.

분석 항목내용
색인 (Index)
  • 해당 URL이 네이버 검색 색인에 포함되어 있는지 확인
  • 최종 수집 일시 및 HTTP 상태 코드 표시
  • 수집 제한 또는 색인 제외 사유 표시
Meta
  • <title> 태그 내용 확인
  • <meta name="description"> 내용 확인
  • robots 메타태그(index/noindex, follow/nofollow) 확인
  • 표준 URL(Canonical) 설정 확인
SEO
  • 이미지 alt 속성 누락 여부
  • 헤딩 태그(h1~h6) 구조 적절성
  • 링크 상태(정상/깨짐)
  • 모바일 최적화 상태
  • 페이지 로딩 속도 관련 권장사항
ℹ️
URL 검사 활용법

새 페이지를 발행했거나 중요한 페이지를 수정한 후 URL 검사를 실행하여 예상대로 색인되고 있는지 확인하세요. 문제가 발견되면 즉시 조치할 수 있습니다.

5. 콘텐츠 노출 및 클릭 리포트

웹마스터도구는 사이트의 검색 성과를 분석하는 콘텐츠 노출 및 클릭 리포트를 제공합니다. 이를 통해 네이버 검색 결과에서 사이트의 성과를 정량적으로 측정할 수 있습니다.

주요 지표

지표설명
노출수 (Impressions)사이트의 페이지가 네이버 검색 결과에 노출된 횟수
클릭수 (Clicks)검색 결과에서 사이트의 페이지를 클릭한 횟수
CTR (Click-Through Rate)클릭수 ÷ 노출수 × 100 (%). 검색 결과에서의 클릭률을 나타냄

보고서 기준

  • 기준일: 최근 90일간의 데이터를 제공합니다.
  • 검색 키워드 TOP 30: 사이트를 검색 결과에 노출시킨 주요 키워드 상위 30개를 확인할 수 있습니다.
  • 일별·주별·월별 추이 그래프를 통해 성과 변화를 모니터링할 수 있습니다.

💡 리포트 활용 전략

  • 노출수는 높지만 CTR이 낮은 키워드 → 제목과 설명 최적화 필요
  • CTR은 높지만 노출수가 낮은 키워드 → 해당 키워드 관련 콘텐츠 확장 고려
  • 주기적으로 리포트를 확인하여 트렌드 변화에 대응하세요

6. 사이트 진단

웹마스터도구의 사이트 진단 기능은 사이트 전반의 검색엔진 최적화 상태를 종합적으로 분석하고 개선점을 제시합니다.

진단 항목

진단 항목설명
색인 (Index)사이트의 전체 페이지 중 네이버 검색 색인에 포함된 비율과 상태
수집제한검색로봇이 수집하지 못한 페이지와 그 사유 (robots.txt 차단, 서버 오류 등)
색인제외색인에서 제외된 페이지와 그 사유 (noindex, 중복 콘텐츠 등)
SEO메타태그, 헤딩 구조, 이미지 alt, 링크 상태 등 SEO 요소 종합 평가

진단 이력 그래프

사이트 진단 결과는 시계열 그래프로 제공되어 시간에 따른 변화를 추적할 수 있습니다. 특정 조치(사이트맵 제출, 콘텐츠 대규모 업데이트 등) 이후 색인 비율이나 수집 제한이 어떻게 변화하는지 모니터링할 수 있습니다.

사이트 진단 FAQ

사이트 진단은 정기적으로 자동 실행됩니다. 또한 웹마스터도구에서 수동으로 진단을 요청할 수도 있습니다.
진단 결과는 참고용 정보입니다. 진단 결과 자체가 랭킹에 직접 영향을 주지는 않지만, 진단에서 발견된 문제를 해결하면 검색엔진 최적화에 도움이 됩니다.
네이버의 검색엔진 최적화 가이드라인을 기준으로 각 항목을 평가합니다. 항목별로 통과/경고/실패 상태가 표시되며, 개선이 필요한 부분에 대한 구체적인 가이드가 제공됩니다.

7. 사이트 최적화 리포트

웹마스터도구는 사이트의 SEO 최적화 상태를 종합적으로 평가하는 최적화 리포트를 제공합니다. 다음 네 가지 항목으로 나누어 분석 결과를 보여줍니다.

검색 로봇 수집

네이버 검색로봇(Yeti)의 사이트 방문 빈도, 수집 성공률, 수집 소요 시간 등을 분석합니다. 수집 비율이 낮은 경우 서버 응답 속도나 robots.txt 설정을 점검해야 합니다.

  • 수집 요청 수 대비 실제 수집 성공 비율
  • 페이지별 수집 소요 시간
  • HTTP 오류 응답 비율

콘텐츠 관리

페이지의 <title>과 <meta name="description"> 설정 상태를 분석합니다. 중복되거나 누락된 경우, 또는 적절하지 않은 길이의 경우 경고를 표시합니다.

  • title 누락·중복·길이 검사
  • description 누락·중복·길이 검사
  • 적정 title 길이: 15~60자
  • 적정 description 길이: 70~160자

사이트 구조

HTML 태그 사용의 적절성, 앵커 텍스트(링크 텍스트)의 명확성, 이미지 alt 속성 등을 분석합니다.

  • 헤딩 태그(h1~h6) 사용 구조
  • 앵커 텍스트의 설명력 (""클릭"" 같은 모호한 텍스트 경고)
  • 이미지 alt 속성 누락 여부
  • HTML 마크업 오류

사이트 활성화

사이트의 전반적인 활성화 상태를 평가합니다. 콘텐츠 업데이트 빈도, 사이트맵 제출 상태, RSS 피드 활성화 여부 등을 확인합니다.

  • 최근 업데이트된 콘텐츠 비율
  • 사이트맵 제출 및 처리 상태
  • RSS 피드 갱신 주기
  • 외부 링크(백링크) 현황

8. 수집 현황

웹마스터도구의 수집현황 메뉴에서는 검색로봇 Yeti가 사이트를 수집한 결과를 상세히 확인할 수 있습니다. 오류 유형별로 수집 결과를 분석하여 문제를 진단할 수 있습니다.

오류 유형 표

오류 유형HTTP 상태설명조치 방법
로봇 비허용- robots.txt에서 Yeti 로봇의 접근이 차단된 경우 robots.txt 수정하여 필요한 경로 허용
DNS 오류- 도메인 이름을 IP 주소로 변환할 수 없는 경우 DNS 설정 확인, 도메인 업체에 문의
연결 실패- 서버에 연결할 수 없는 경우 (서버 다운, 방화벽 차단 등) 서버 상태 확인, 방화벽 설정 점검
서버 오류5xx 서버 내부 오류로 페이지를 제공할 수 없는 경우 서버 로그 확인, 웹서버 설정 점검
페이지 없음404 요청한 페이지가 서버에 존재하지 않는 경우 깨진 링크 수정 또는 301 리다이렉트 설정
권한 없음403 서버에서 접근 권한을 거부한 경우 디렉토리 권한 설정 확인
리다이렉트 초과- 리다이렉트가 너무 많이 이루어져 최종 페이지에 도달하지 못한 경우 리다이렉트 체인 단순화
시간 초과- 서버 응답이 설정된 시간을 초과한 경우 서버 응답 속도 개선, 페이지 용량 최적화

⚠️ 오류 발견 시 대응

수집 현황에서 특정 오류 유형이 급증한 경우, 서버 장애나 사이트 구조 변경 문제를 의심해야 합니다. 정기적으로 모니터링하여 신속히 대응하세요.

9. IndexNow 프로토콜

소개

IndexNow는 웹사이트의 콘텐츠 변경을 검색엔진에 즉시 알리는 프로토콜입니다. 네이버를 포함한 여러 검색엔진(Bing, Yandex 등)이 지원합니다. 사이트맵 제출보다 더 빠른 색인 갱신이 가능하며, 새로운 페이지 발행이나 기존 페이지 수정 시 검색엔진이 신속하게 반영할 수 있도록 돕습니다.

💡 IndexNow의 장점

  • URL 변경을 실시간에 가깝게 알릴 수 있음
  • 검색엔진이 크롤링 예약을 기다릴 필요 없음
  • 단일 API로 여러 검색엔진에 동시 전송 가능
  • 사이트맵보다 효율적인 색인 갱신

API Key 생성 규칙

IndexNow를 사용하려면 API Key가 필요합니다. API Key는 다음 규칙을 따라야 합니다.

항목규칙
길이8~128자
형식16진수(hex) 문자열 (0-9, a-f, A-F)
파일 위치https://yourdomain.com/{api-key}.txt 형태로 접근 가능해야 함
 
# API Key 예시 (64자)
7a8b9c0d1e2f3a4b5c6d7e8f9a0b1c2d3e4f5a6b7c8d9e0f1a2b3c4d5e6f7a8

단일 페이지 갱신 요청 (GET)

하나의 URL을 갱신하려면 GET 요청을 사용합니다.

GET https://searchadvisor.naver.com/indexnow?url=URL&key=APIKEY

# 예시
GET https://searchadvisor.naver.com/indexnow?url=https://example.com/new-page&key=7a8b9c0d...
# cURL로 요청하는 예시
curl -X GET "https://searchadvisor.naver.com/indexnow?url=https://example.com/new-page&key=7a8b9c0d1e2f3a4b5c6d7e8f9a0b1c2d3e4f5a6b7c8d9e0f1a2b3c4d5e6f7a8"

다중 페이지 갱신 요청 (POST)

여러 URL을 한 번에 갱신하려면 POST 요청을 사용합니다. 최대 10,000개의 URL을 한 번에 전송할 수 있습니다.

POST https://searchadvisor.naver.com/indexnow
Content-Type: application/json

{
  "urlList": [
    "https://example.com/page1",
    "https://example.com/page2",
    "https://example.com/page3"
  ],
  "key": "7a8b9c0d1e2f3a4b5c6d7e8f9a0b1c2d3e4f5a6b7c8d9e0f1a2b3c4d5e6f7a8"
}
# cURL로 POST 요청하는 예시
curl -X POST "https://searchadvisor.naver.com/indexnow" \
  -H "Content-Type: application/json" \
  -d '{
    "urlList": [
      "https://example.com/page1",
      "https://example.com/page2"
    ],
    "key": "7a8b9c0d1e2f3a4b5c6d7e8f9a0b1c2d3e4f5a6b7c8d9e0f1a2b3c4d5e6f7a8"
  }'

HTTP 응답 코드

응답 코드의미설명
200 OK성공요청이 정상적으로 처리되었습니다.
202 Accepted접수 완료요청이 접수되었으며, 비동기적으로 처리됩니다.
400 Bad Request잘못된 요청URL 형식이 올바르지 않거나 필수 파라미터가 누락되었습니다.
403 Forbidden권한 없음API Key가 유효하지 않거나 인증에 실패했습니다.
422 Unprocessable Entity처리 불가요청 본문의 형식이 올바르지 않습니다 (POST 요청 시).
429 Too Many Requests요청 초과일일 요청 한도를 초과했습니다. 잠시 후 다시 시도하세요.
500 Internal Server Error서버 오류서버 내부 오류가 발생했습니다. 나중에 다시 시도하세요.

IndexNow FAQ

사이트맵은 검색엔진이 정기적으로 방문하여 확인하는 방식이지만, IndexNow는 변경이 발생한 즉시 검색엔진에 알릴 수 있습니다. IndexNow는 사이트맵을 대체하는 것이 아니라 보완하는 도구입니다. 두 가지를 함께 사용하는 것이 가장 효과적입니다.
네이버는 IndexNow API에 대해 일일 요청 한도를 적용할 수 있습니다. 429 응답 코드를 받으면 일정 시간 후에 다시 시도해야 합니다. 또한 POST 요청 시 한 번에 최대 10,000개의 URL을 전송할 수 있습니다.
API Key는 직접 생성할 수 있습니다. 8~128자 길이의 16진수 문자열(0-9, a-f, A-F)을 만들어 사용하면 됩니다. 예를 들어 openssl rand -hex 32 명령어로 64자의 안전한 키를 생성할 수 있습니다. 생성한 키는 https://yourdomain.com/{api-key}.txt 파일로 호스팅해야 합니다.
IndexNow 프로토콜을 지원하는 검색엔진들은 각자의 IndexNow 엔드포인트를 가지고 있습니다. 네이버의 엔드포인트는 https://searchadvisor.naver.com/indexnow입니다. 하지만 IndexNow 제휴 검색엔진 중 하나에 URL을 제출하면, 제휴된 다른 검색엔진에도 공유될 수 있습니다.
IndexNow 요청이 접수(202 Accepted)되면 검색엔진이 해당 URL을 우선적으로 크롤링합니다. 그러나 모든 검색엔진이 동일한 속도로 처리하는 것은 아니며, 검색엔진의 정책과 현재 부하에 따라 실제 반영 시간은 달라질 수 있습니다.
네, IndexNow를 사용하려면 네이버 웹마스터도구(Search Advisor)에 사이트를 등록하고 소유권을 인증해야 합니다. API Key 인증과 함께 사이트 소유권 확인이 필요합니다.

자주 묻는 질문 (FAQ)

검색결과편집은 네이버 웹마스터도구에서 제공하는 기능으로, 사이트의 검색 결과 스니펫(제목, 설명, URL)을 직접 수정할 수 있는 기능입니다. 검색 결과에 노출되는 제목과 설명을 원하는 대로 편집하여 CTR을 향상시킬 수 있습니다.
다음 사항을 순서대로 확인하세요.
  1. 웹마스터도구에 사이트가 등록되어 있고 소유권이 인증되었는지 확인
  2. site:yourdomain.com으로 검색하여 색인 상태 확인
  3. robots.txt에서 Yeti 로봇이 차단되지 않았는지 확인
  4. noindex 메타태그가 설정되지 않았는지 확인
  5. 사이트맵과 RSS를 웹마스터도구에 제출
  6. 수집요청 기능으로 URL 직접 요청
검색 결과에서 페이지를 삭제하는 방법은 다음과 같습니다.
  1. 일시적 삭제: 웹마스터도구의 검색삭제요청 기능 사용 (캐시 삭제)
  2. 영구적 삭제: 해당 페이지에 <meta name="robots" content="noindex"> 추가
  3. URL 삭제: 페이지 자체를 410(Gone) 또는 404(Not Found)로 응답
  4. robots.txt로 차단 (단, 이미 색인된 페이지는 robots.txt만으로 삭제되지 않음)
서브링크는 네이버 검색 결과에서 사이트의 주요 페이지 링크가 추가로 표시되는 기능입니다. 직접 설정할 수 없으며, 네이버 검색엔진이 사이트 구조를 분석하여 자동으로 생성합니다. 서브링크 노출 가능성을 높이려면:
  • 명확하고 일관된 사이트 구조(네비게이션) 구축
  • 각 섹션의 페이지에 적절한 <title>과 헤딩 태그 사용
  • 사이트맵을 통해 전체 페이지 구조를 검색엔진에 전달
  • 사이트 내부 링크 구조를 논리적으로 구성
사이트 등록 후 검색 결과에 반영되는 시간은 사이트의 상태와 콘텐츠 품질에 따라 다릅니다. 일반적으로 웹마스터도구에 등록하고 사이트맵을 제출하면 1~2주 내에 검색 결과에 나타나기 시작합니다. 사이트가 이미 다른 검색엔진에 잘 노출되고 있다면 더 빠를 수 있습니다.
