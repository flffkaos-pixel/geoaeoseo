⚙️ 기술적 SEO (Technical SEO)

사이트가 검색엔진에 의해 잘 크롤링되고 색인되는지 확인합니다.

  • HTTPS 보안 인증서 설치 및 모든 페이지에 HTTPS 적용
  • HTTP → HTTPS 301 리디렉션 설정
  • robots.txt 파일 작성 및 최적화
  • XML 사이트맵 생성 및 Search Console 제출
  • 사이트맵 인덱스 파일 사용 (대규모 사이트)
  • 중복 페이지에 canonical 태그 설정
  • 404 에러 페이지 최소화 및 커스텀 404 페이지
  • 301/302 리디렉션 체인 제거
  • JavaScript SEO 베스트 프랙티스 준수
  • 모바일 반응형 디자인 적용
  • 페이지 로드 속도 2.5초 이하 (LCP)
  • INP 200ms 이하 유지
  • CLS 0.1 이하 유지
  • 이미지 lazy loading 적용
  • 이미지 WebP/AVIF 포맷 사용
  • CSS/JS 파일 minify 및 번들링
  • CDN 사용으로 전송 속도 향상
  • URL 구조 단순화 (의미 있는 URL)
  • 구조화된 내부 링크 (3클릭 이내 도달)
  • hreflang 태그 (다국어 사이트)

📝 온페이지 SEO (On-Page SEO)

각 페이지의 메타데이터와 콘텐츠 최적화입니다.

  • 모든 페이지에 고유한 title 태그 (50-60자)
  • 모든 페이지에 매력적인 meta description (150-160자)
  • H1 태그가 페이지당 1개, 핵심 키워드 포함
  • 논리적인 H2 → H3 → H4 헤딩 계층 구조
  • 시맨틱 HTML5 태그 사용 (article, section, nav, aside)
  • 모든 이미지에 설명적인 alt 텍스트
  • 파일명이 의미 있는 이미지 (예: seo-guide.png)
  • 내부 링크에 의미 있는 앵커 텍스트
  • 외부 링크는 rel="noopener" 속성 사용
  • 키워드 스터핑 없이 자연스러운 언어 사용
  • Open Graph 태그 (소셜 공유)
  • Twitter Card 태그
  • Canonical URL 설정
  • 메타 robots 태그 적절히 사용
  • 관련성 높은 페이지 간 클러스터 구조

📚 콘텐츠 (Content)

Google이 사랑하는 고품질, 사람 중심 콘텐츠를 만듭니다.

  • 각 콘텐츠에 명확한 검색 의도 충족
  • 독자 페르소나 정의 및 맞춤 콘텐츠
  • 1인칭 경험과 디테일 포함
  • 차별화된 관점과 독창성
  • Commodity 콘텐츠 지양 (일반적 정보 회피)
  • 작성자 정보 명시 (작성자 페이지 링크)
  • 작성/수정 날짜 표시
  • 출처 및 인용 명시
  • 통계와 데이터로 주장 강화
  • 시각 자료 활용 (이미지, 차트, 비디오)
  • 충분한 콘텐츠 길이 (얕은 콘텐츠 회피)
  • 정기적인 콘텐츠 업데이트 및 개선
  • 오래된 콘텐츠 리프레시
  • 사용자 의도에 맞는 콘텐츠 형식 (리스트, 가이드, 비교 등)
  • 스팸 정책 위반 요소 없음 (자동 생성, 키워드 스터핑 등)

✅ E-E-A-T (경험, 전문성, 권위성, 신뢰성)

Google이 평가하는 콘텐츠 품질의 4가지 핵심 요소입니다.

  • Experience: 작성자의 직접 경험이 드러남
  • Expertise: 해당 분야 전문 지식 표현
  • Authoritativeness: 외부에서 권위 있는 출처로 인정
  • Trustworthiness: HTTPS, 투명한 정보, 연락처
  • 상세한 작성자 소개 페이지
  • 작성자의 자격/경력 명시
  • 검토자 또는 편집자 정보
  • 회사 About 페이지 (팀, 미션, 역사)
  • 명확한 Contact 페이지 (전화, 이메일, 주소)
  • 정책 페이지 (Privacy, Terms, Cookies)
  • 명확한 편집 정책
  • YMYL 콘텐츠는 전문가가 작성/검토
  • 광고/후원 콘텐츠 명확한 구분
  • 업계 수상, 인증, 멤버십 표시
  • 고객 후기 및 사례 연구

🔖 구조화 데이터 (Schema.org)

Schema.org JSON-LD로 리치 결과에 노출되도록 합니다.

  • Article 스키마 (블로그 포스트)
  • BreadcrumbList 스키마 (모든 페이지)
  • Organization 스키마 (회사 정보)
  • WebSite 스키마 (사이트 전체)
  • FAQPage 스키마 (FAQ 콘텐츠)
  • HowTo 스키마 (단계별 가이드)
  • Product 스키마 (제품 페이지)
  • LocalBusiness 스키마 (오프라인 매장)
  • Review/AggregateRating 스키마 (해당 시)
  • Speakable 스키마 (음성 검색)
  • VideoObject 스키마 (비디오)
  • 구조화 데이터가 페이지의 보이는 콘텐츠 반영
  • Rich Results Test로 마크업 검증
  • Schema Markup Validator로 표준 준수 확인

🤖 GEO (생성형 엔진 최적화)

AI 검색 엔진에서 콘텐츠가 인용되도록 최적화합니다.

  • 독보적인 관점과 1인칭 경험 콘텐츠
  • Commodity 콘텐츠 지양
  • 통계/수치/출처 명시
  • 엔티티 명확성 (Schema.org 활용)
  • 권위 있는 출처에서 반복 인용/언급
  • 위키피디아 등재 시도 (해당 시)
  • 산업 디렉토리 등록
  • 작성/수정 날짜 명시 (최신성 신호)
  • 정기적인 콘텐츠 업데이트
  • 명확한 답변-배경-상세 구조 (BLUF)
  • 사실 정보에 출처 링크
  • llms.txt는 만들지 않음 (Google 공식 입장)
  • AI 크롤러 허용 (인용 기회)
  • 청킹(불필요한 콘텐츠 분할) 안 함
  • AI 가시성 추적 도구 활용

💡 AEO (답변 엔진 최적화)

Featured Snippet, 음성 검색, AI 답변에서 선택받도록 합니다.

  • 질문형 H2/H3 헤딩 사용
  • 질문 직후 40-60단어 답변 배치
  • BLUF (Bottom Line Up Front) 구조
  • 리스트와 표 적극 활용
  • FAQPage 스키마 마크업
  • HowTo 스키마 마크업
  • People Also Ask 질문에 대한 답변 포함
  • Speakable 스키마 (음성 검색)
  • 시맨틱 HTML5 태그
  • Featured Snippet 경쟁 페이지 분석
  • 로컬 SEO 최적화 (해당 시)
  • Google Business Profile 최적화 (해당 시)
  • 콘텐츠 명확성과 간결성
  • 모바일 가독성 확보

🕷️ AI 크롤러 관리

AI 크롤러 정책을 결정하고 robots.txt로 관리합니다.

  • robots.txt에 주요 AI 크롤러 정책 명시
  • GPTBot 정책 결정 (학습 허용/차단)
  • OAI-SearchBot 허용 (인용 기회)
  • ClaudeBot 정책 결정
  • CCBot 정책 결정 (대부분 LLM 학습 소스)
  • PerplexityBot 허용 (인용 기회)
  • Google-Extended 결정 (제미나이 학습)
  • Applebot-Extended 결정 (Apple Intelligence)
  • 유료/독점 콘텐츠는 AI 크롤러 차단
  • Google-Extended는 메타 태그로도 제어 가능
  • AI 정책 페이지 작성 (투명성)
  • 주기적으로 AI 트래픽 모니터링
  • robots.txt 테스터로 검증

📊 분석 및 측정

데이터 기반으로 SEO/GEO/AEO 성과를 추적합니다.

  • Google Search Console 설치 및 인증
  • Google Analytics 4 (GA4) 설치
  • Search Console과 GA4 연동
  • 사이트맵 Search Console 제출
  • URL Inspection 도구 활용
  • Core Web Vitals 모니터링
  • 키워드별 순위 추적
  • 오가닉 트래픽 트렌드 분석
  • Featured Snippet 점유율 추적
  • AI 가시성 추적 도구 활용
  • AI 트래픽 GA4 필터링
  • 전환율 분석 (오가닉 vs AI 유입)
  • 월간/분기별 리포트 작성
  • 경쟁사 벤치마킹
  • Google 알고리즘 업데이트 모니터링

🔗 오프페이지 SEO

외부 신호로 사이트의 권위성을 높입니다.

  • 품질 높은 사이트에서 백링크 확보
  • 백링크 구매/판매 안 함 (스팸 정책)
  • 게스트 포스팅 (관련성 있는 사이트)
  • 디지털 PR 활동 (보도자료, 언론 인용)
  • 브랜드 멘션 모니터링
  • 업계 인플루언서 네트워크 구축
  • 소셜 미디어 프로필 관리
  • 산업 디렉토리/리뷰 사이트 등록
  • Reddit, Quora 등 커뮤니티 활동 (스팸 아닌)
  • 브로큰 백링크 모니터링 및 복구
  • 스팸성 백링크 disavow (필요 시)
  • 로컬 인용 일관성 (NAP: Name, Address, Phone)
